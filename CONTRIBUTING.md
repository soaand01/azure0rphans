## How to Contribute

### Pull Request Process

**All contributions must be submitted via Pull Request (PR).** Direct pushes to the main branch are not accepted.

1. **Fork the Repository**
   ```bash
   # Fork the repo on GitHub, then clone your fork
   git clone https://github.com/YOUR-USERNAME/azure0rphans.git
   cd azure0rphans
   ```

2. **Create a Feature Branch**
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/your-bug-fix
   ```

3. **Make Your Changes**
   - Write clean, readable code
   - Follow existing code style
   - Test your changes locally
   - Update documentation if needed

4. **Commit Your Changes**
   ```bash
   git add .
   git commit -m "Clear description of your changes"
   ```

5. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Open a Pull Request**
   - Go to the original repository on GitHub
   - Click "New Pull Request"
   - Select your fork and branch
   - Provide a clear description of your changes
   - Link any related issues

## Contribution Guidelines

### Code Quality
- Ensure your code runs without errors
- Test in both demo mode and production mode (if applicable)
- Keep dependencies minimal

### Documentation
- Update README.md if you add new features
- Add comments for complex logic
- Update requirements.txt if adding dependencies

### Pull Request Requirements
- **Clear title and description** - explain what and why
- **One feature per PR** - keep PRs focused
- **Test your changes** - ensure nothing breaks
- **Reference issues** - link related issues if applicable
